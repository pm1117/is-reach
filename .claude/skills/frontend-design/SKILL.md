# Frontend Design Skill

is-reach の PC 向け管理画面の UI/UX 方針を策定する。

## 原則

- **実装コードは原則書かない**（方針・仕様まで）。UI 実装 PR を担当する場合は、その旨をユーザーが明示したときのみ例外とする。
- 成果物は `docs/ui-spec.md`。人間の承認を得てから実装フェーズへ渡す。
- PC 向け管理画面が対象。モバイル対応は要件で明示されない限りスコープ外。

## 成果物（docs/ui-spec.md）に含めるもの

- 画面一覧と画面遷移
- 主要 UI コンポーネントの方針（高忠実度コンポーネントの粒度・命名）
- Tailwind CSS の利用方針（デザイントークン、色・タイポグラフィ・余白のスケール）
- 状態表現（ローディング / 空 / エラー / 権限なし）
- データ取得 → 分析 → メッセージ生成の各フローに対応する画面の責務

## 設計時の観点

- インサイドセールス担当者の業務フロー（リスト確認 → 分析結果確認 → 生成文の確認・編集 → 送信判断）に沿った情報設計にする
- LLM 生成文は**必ず人間が確認・編集できる UI** を前提にする（自動送信を既定にしない）
- 外部サイト由来のテキスト（スクレイピング結果）を表示する箇所は、エスケープ・サニタイズ前提であることを仕様に明記する
