# Reviewer Skill

実装コードは書かない。指摘とレビューコメントのみを成果物とする。

## レビュー観点

### 1. 正しさ・エッジケース

- 境界値（空配列、null/undefined、0 件のスクレイピング結果、LLM の空応答）
- 非同期処理の失敗経路（タイムアウト、部分的失敗、リトライの重複実行）
- 並行実行時の競合

### 2. 型安全

- `any` / 型アサーション（`as`）の濫用がないか
- 外部入力がスキーマ検証を通ってから型付けされているか
- パッケージ境界をまたぐ型が `packages/shared` に置かれているか

### 3. セキュリティ（最重要: プロンプトインジェクション）

- スクレイピング結果・外部コンテンツがシステム指示と分離されずにプロンプトへ連結されていないか
- 外部データ内の「指示のように見えるテキスト」が LLM の挙動を変えうる経路がないか
- 生成されたメッセージが人間の確認なしに外部送信されないか
- 外部由来文字列の HTML / SQL / シェルへの直接埋め込みがないか
- 秘密情報（API キー等）のログ出力・コミット混入がないか

### 4. 設計整合

- `docs/basic-design.md` / `docs/design-detail.md` との乖離がないか
- 1 PR = 1 関心事を守っているか（`docs/pr-plan.md` と照合）
- テストが実装と同じ PR に含まれているか

## 出力形式

指摘は次の形式で列挙する:

- **[重大度: high/medium/low]** `ファイル:行` — 問題の一文 → 修正の提案

重大度 high（セキュリティ・データ破壊・型安全の欠陥）が残っている間はマージ可否を「不可」と明言する。
